# WBS Builder — 建置計畫書 v1.0

> 目標：提供工程師一個**離線可用、單一 HTML 檔**的 WBS（工作分解結構）動態選單工具，
> 無安裝需求，適用於氣隙（air-gapped）生產環境。

---

## 一、專案背景與目標

| 項目 | 說明 |
|------|------|
| 使用情境 | 內部工程師快速建立專案 WBS，不依賴外部服務 |
| 部署限制 | 氣隙網路，不可使用 CDN / npm 套件，需為單一 `.html` 檔 |
| 主要使用者 | 專案負責人、技術 lead、BA |
| 開發工具 | Claude Code（本地 CLI）+ 瀏覽器直接預覽 |

---

## 二、功能範圍（Scope）

### 第 1 階段 — 核心編輯（Prototype 已完成）

- [x] 樹狀結構展示（支援最多 5 層）
- [x] 新增同層節點 / 新增子節點
- [x] 雙擊行內編輯節點標題
- [x] 拖曳排序（同層重排）
- [x] 展開 / 折疊子樹
- [x] 自動 WBS 編號（1, 1.1, 1.1.1…）
- [x] 匯出 CSV（含 BOM，Excel 可直接開啟）
- [x] 匯出 / 匯入 JSON（存檔續編）
- [x] AI 輔助生成（需連線，氣隙環境可移除）

### 第 2 階段 — 強化功能（Claude Code 繼續開發）

- [ ] `localStorage` 自動存檔，重開瀏覽器不遺失
- [ ] 多專案管理（切換 / 新建 / 刪除專案）
- [ ] 節點備註欄位（展開顯示說明文字）
- [ ] 節點狀態標籤（未開始 / 進行中 / 完成）
- [ ] 跨層拖曳（移動節點到不同父節點下）
- [ ] 鍵盤快捷鍵（Tab 縮排、Shift+Tab 反縮排、Delete 刪除）
- [ ] 復原 / 重做（Ctrl+Z / Ctrl+Y）

### 第 3 階段 — 進階輸出（選用）

- [ ] 匯出 Excel `.xlsx`（使用 SheetJS inline bundle）
- [ ] 列印 / 匯出 PDF（瀏覽器列印樣式）
- [ ] 甘特圖預覽（輕量 SVG 渲染）

---

## 三、技術決策

| 項目 | 決策 | 理由 |
|------|------|------|
| 框架 | 純 Vanilla JS | 無外部依賴，離線可用 |
| 樣式 | 內嵌 CSS + CSS Variables | 主題一致性，無需預處理器 |
| 資料格式 | JSON 樹狀結構 | 易於序列化、匯入匯出 |
| 存檔 | `localStorage`（第 2 階段） | 瀏覽器原生，無伺服器需求 |
| 匯出 Excel | SheetJS CDN → inline bundle | 氣隙環境需先 bundle 進 HTML |
| AI 生成 | Anthropic API（可選） | 連網機器可用；氣隙環境移除按鈕 |

## 四、檔案結構（Claude Code 開發版）

```
wbs/
├── index.html          # 開發版（拆分檔案）
├── src/
│   ├── app.js          # 主邏輯
│   ├── tree.js         # 樹狀結構工具函式
│   ├── render.js       # DOM 渲染
│   ├── export.js       # CSV / JSON / XLSX 匯出
│   └── ai.js           # AI 生成（可移除）
├── styles/
│   └── main.css
├── dist/
│   └── wbs-builder.html  # 打包後的單一離線 HTML
├── CLAUDE.md           # Claude Code 工作指引
└── README.md
```

## 五、CLAUDE.md 建議內容（Claude Code 工作指引）

```markdown
# WBS Builder — Claude Code 工作指引

## 專案目標
單一離線 HTML 檔，無外部依賴，適用氣隙環境。

## 開發規則
- 所有邏輯寫在 src/ 下，最終用 build script 合併成 dist/wbs-builder.html
- 禁止引入任何 CDN 或 npm 套件（除非手動 inline）
- CSS 使用 CSS Variables，色彩定義集中在 :root
- ID 命名：kebab-case；JS 函式：camelCase

## 第 2 階段優先任務
1. localStorage 自動存檔
2. 跨層拖曳
3. 鍵盤快捷鍵（Tab / Shift+Tab / Ctrl+Z）

## 已知限制
- 拖曳目前只支援同層重排，跨層移動待實作
- AI 生成按鈕在氣隙環境應隱藏或移除
- IE 不支援（不需考慮）
```

## 七、已知限制

- 拖曳目前為同層重排；**跨層拖曳**為第 2 階段目標
- AI 生成需要網路，氣隙環境請移除 `#ai-panel` 及相關 JS
- 大型專案（500+ 節點）未做效能最佳化，可在第 2 階段視需求加入虛擬捲動
- 不支援多人同時編輯（非設計目標）
