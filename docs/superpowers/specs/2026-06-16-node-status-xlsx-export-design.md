# 節點狀態標籤 + XLSX 匯出 — 設計文件

> 日期：2026-06-16
> 狀態：已確認，待轉換為實作計畫

## 背景

`CLAUDE.md` 第 2 階段待辦清單中，使用者決定排除「localStorage 自動存檔」「鍵盤快捷鍵」「多專案管理」「列印 PDF」四項，先規劃實作其餘項目。本文件涵蓋第一個子專案：**節點狀態標籤**，並依使用者要求一併納入**XLSX 匯出**（原列於第 3 階段，因與狀態標籤的 CSV 欄位調整高度相關而合併規劃）。

## 一、資料模型

節點物件新增欄位：

```js
{ id, title, children, owner?, startDate?, endDate?, status? }
// status: '未開始' | '進行中' | '完成'
```

- 直接以中文字串儲存，與現有 `owner`/`startDate`/`endDate` 的儲存風格一致，不另建 enum 對照層。
- 新節點建立時（`addRoot`/`addSibling`/`addChildNode`）明確賦值 `status: '未開始'`（與 owner/date 的「留空即未設定」不同，狀態一律有預設值，確保畫面一律可見徽章）。
- `injectIds(nodes)`（套用模板、JSON 匯入）若來源節點缺少 `status`，補上預設值 `'未開始'`，向下相容舊存檔。

## 二、狀態徽章 UI

**位置**：`.node-row` 中 `level-dot` 之後、`node-title` 之前。

**樣式**（對應既有 `:root` 色彩變數）：

| 狀態 | 圖示 | 顏色變數 |
|------|------|---------|
| 未開始 | `○` | `--muted` |
| 進行中 | `◐` | `--yellow` |
| 完成 | `●` | `--green` |

徽章 `title` 屬性顯示完整文字（如「進行中」），供 hover tooltip。手機版（`@media max-width:640px`）徽章常駐顯示，不受 `.node-actions` 的 hover-only 限制影響（徽章本身不在 `.node-actions` 內，預期不受影響，但需在實作時確認排版不跑版）。

**互動**：點擊徽章彈出小選單（3 個選項，各帶色點＋文字），點選任一選項套用新狀態並關閉選單；點擊選單外部則關閉且不變更。需呼叫 `e.stopPropagation()` 避免觸發 `node-row` 的拖曳事件，比照現有 `act-btn`／`meta-field` 的寫法。

## 三、統計區（`#stats`）

新增第 4 個統計項目：「已完成 X/Y（Z%）」。

- 計算方式：`allFlat(tree)` 結果中 `status === '完成'` 的節點數 / 總節點數。
- 邊界情況：總數為 0 時顯示 `0/0（0%）`，避免除以零。

## 四、CSV 匯出

`exportCSV()` 表頭與欄位順序調整為：

```
編號, 層級, 工作項目, 負責人, 狀態, 開始日期, 結束日期
```

`狀態` 欄位值取 `n.status || '未開始'`。

## 五、XLSX 匯出（新功能）

**UI**：Header 新增「⬇ XLSX」按鈕，與現有「⬇ CSV/Excel」「⬇ JSON」「⬆ 匯入」並列，不取代既有按鈕。

**內容**：欄位與 CSV 完全相同（`編號, 層級, 工作項目, 負責人, 狀態, 開始日期, 結束日期`），不做縮排、上色等樣式處理（求正確可用，樣式留待未來迭代）。

**技術方案**：
- 需手動將 SheetJS（`xlsx.full.min.js`，壓縮版約數百 KB）原始碼 inline 進 `index.html` 的 `<script>` 區塊，符合 `CLAUDE.md` 「禁止 CDN／npm，第三方套件需手動 inline」的限制。
- 實作階段需先一次性取得該函式庫原始碼內嵌；之後完全離線可用，不需網路。
- 新增 `exportXLSX()` 函式：邏輯與 `exportCSV()` 並行，共用列資料組裝（可抽出共用的 rows 產生邏輯），輸出改用 `XLSX.utils.aoa_to_sheet` + `XLSX.writeFile`。

**已知影響**：單一 HTML 檔案大小會明顯增加（SheetJS 函式庫體積），此為使用者已知並接受的取捨。

## 六、邊界情況

- 空樹（無任何節點）：CSV／XLSX 皆只輸出表頭列；統計區完成比例顯示 `0/0（0%）`。
- 舊版 JSON 匯入（無 `status` 欄位）：經 `injectIds()` 補預設值 `'未開始'`，不會中斷匯入流程。

## 七、測試方式

專案無自動化測試（單一離線 HTML，無建置流程），沿用既有手動驗證慣例：

1. 建立節點，依序點擊狀態徽章切換三種狀態，確認顏色與選單顯示正確
2. 確認統計區「已完成」比例隨狀態變化即時更新
3. 套用模板／匯入舊版 JSON（無 status 欄位），確認自動補上「未開始」且不報錯
4. 匯出 CSV，以 Excel 開啟確認「狀態」欄位與中文顯示正常
5. 匯出 XLSX，以 Excel 開啟確認檔案可正常開啟、欄位與 CSV 一致
6. 手機版（≤640px）檢查狀態徽章與選單排版不跑版
